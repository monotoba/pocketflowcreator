# Contributing

This project should evolve conservatively.

## Development rules

1. Prefer refactoring over rewrites.
2. Keep generated code separate from user-owned custom code.
3. Add or update tests for every behavior change.
4. Keep project metadata deterministic and version-control friendly.
5. Do not store API keys or provider credentials in project files.
6. Keep GUI behavior property-first and PocketFlow-compatible.

## Suggested commit style

```text
Add graph validation for unresolved action ports
Refactor node type metadata loader
Add smoke tests for generated flows
Document custom node inheritance model
```
